#1. Using ng-template elements in Angular 2 Content of templates are not rendered on its own directly
Content written in ng-template are rendered as it is ...they are not wrapped in ng-template tag in DOM.
to show content of ng-template we need to use a template ref & then use ViewChild to query and get reference to the template & then embed the template using ViewContainerRef

#3. Content Projection - < ng-content > with multiple DOM elements in different location in the DOM it allows us to inject content somewhere else...... by default ng-content will replace all the content passed to it in one slot. In this example, h3 tag is injected into the auth-form in place of the ng-content tag
but if we want to inject multiple html element in different place to a component we need to explicitly specify what element comes where using `select` like this--- `< ng-content select="h3"> select="#myId" select=".my-class"`
< ng-content select="my-component-selector">
you can also inject component it could be tag, id or classes
